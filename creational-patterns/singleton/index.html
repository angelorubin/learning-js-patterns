Padrões Javascript

Singleton

A ideia do padrão singleton é ter apenas uma instância de uma classe específica.

Isso significa que na segunda vez que você usar a mesma classe para criar um novo objeto, deverá obter o mesmo objeto que foi criado na primeira vez.

E como isso se aplica ao JavaScript? Em JavaScript não existem classes, apenas objetos. Quando você cria um novo objeto, na verdade não há nenhum outro objeto como ele, e o novo objeto já é um singleton. Criar um objeto simples usando um objeto literal, também é um exemplo de singleton: 

let obj = {
  myprop: "my value",
};

Em JavaScript, os objetos nunca são iguais, a menos que sejam o mesmo objeto, então, mesmo que você crie um objeto idêntico com os mesmos membros, ele não será igual ao primeiro: 

let obj2 = {
	myprop: "my value",
}

obj === obj2; // false

obj == obj2; // false
